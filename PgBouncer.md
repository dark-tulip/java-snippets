## Режимы работы
- session pooling - prepared statements, advisory locks, временные таблицы
- transaction pooling - транзакции
- statement pooling - только отдельные запросы


## Database timeouts:
- authentication_timeout
- wal_sender_timeout
- statement_timeout
- lock_timeout
- idle_in_transaction_session_timeout
- deadlock_timeout
- TCP keep alive
